# How to Publish a Blog Post

Ragnild Web uses Strapi CMS for managing blog content. Publishing a new blog post is done through the Strapi admin interface.

## Prerequisites

Ensure the Strapi CMS is running:

```bash
cd cms
npm run develop
```

Access the admin panel at: [http://localhost:1337/admin](http://localhost:1337/admin)

## 1. Create a New Blog Post

1. Log in to the Strapi admin panel
2. Navigate to **Content Manager** → **Blogs**
3. Click **Create new entry**

## 2. Fill in the Blog Post Details

### Required Fields:

- **Title**: The headline of your article
  - Example: "The Future of Cloud Computing"

- **Slug**: The URL-friendly version of your title (used in the URL)
  - Example: "future-of-cloud-computing"
  - This becomes: `ragnild.com/blog/future-of-cloud-computing`

- **Content**: Your blog post content in rich text format
  - Use the rich text editor to format your content
  - Supports: Bold, italic, headings, links, lists, etc.

- **Publish Date**: The date when the post should be published
  - Format: YYYY-MM-DD
  - Used for sorting posts (newest first)

### Optional Fields:

- **Cover Image** (coverimage): A featured image for your blog post
  - Recommended size: 1200x630 pixels (or similar 16:9 aspect ratio)
  - Formats: JPG, PNG, WebP
  - Upload using the media library or provide a URL
  - This image appears on:
    - Blog listing page (as card thumbnail)
    - Individual blog post page (as hero image)

## 3. Write Your Content

Use the Strapi rich text editor to write your blog post:

### Supported Formatting:

- **Headings**: Available in the editor toolbar
- **Bold/Italic**: Use the formatting buttons
- **Lists**: Ordered and unordered lists available
- **Links**: Use the link button to add hyperlinks
- **Paragraphs**: Automatically created

### Content Tips:

- Keep paragraphs concise and readable
- Use headings to break up long content
- Add links to reference materials
- Write engaging introductions

## 4. Auto-Generated Features

The following are automatically generated by the system:

- **Excerpt**: Auto-generated from the first 150 characters of your content
- **Read Time**: Automatically calculated based on word count (200 words/min)
- **Author**: Defaults to "Ragnild Team" (can be customized in code)
- **Category**: Defaults to "General" (can be customized in code)

## 5. Preview and Publish

1. **Save as Draft**: Click "Save" to save without publishing
2. **Preview**:
   - Save your post
   - Navigate to `http://localhost:3000/blog`
   - Your post will appear if published
3. **Publish**: Click "Publish" to make the post live

## 6. Edit Existing Posts

1. Go to **Content Manager** → **Blogs**
2. Click on the post you want to edit
3. Make your changes
4. Click "Save" or "Publish"

## 7. Delete Posts

1. Go to **Content Manager** → **Blogs**
2. Click on the post you want to delete
3. Click the "Delete" button
4. Confirm deletion

## API Integration

The Next.js frontend automatically fetches blog posts from:
```
http://localhost:1337/api/blogs
```

Posts are:
- Sorted by date (newest first)
- Cached for performance
- Displayed in a responsive grid layout

## Troubleshooting

### Posts Not Showing Up?

1. Ensure Strapi is running on port 1337
2. Check that the post is published (not draft)
3. Verify `NEXT_PUBLIC_STRAPI_API_URL` in `.env.local`
4. Check browser console for API errors

### Content Not Formatting Correctly?

- Use the rich text editor toolbar
- Avoid pasting from Word (can cause formatting issues)
- Use "Clear formatting" if needed

### API Connection Issues?

Check that:
- Strapi is running: `cd cms && npm run develop`
- API URL is correct in `.env.local`
- No CORS errors in browser console
